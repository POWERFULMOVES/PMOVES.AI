# PMOVES.AI Services and Integrations

Below is a comprehensive list of the services and integrations in the PMOVES.AI monorepo (and its git submodules), along with each service’s role, code path, image details, dependencies, ports, health endpoints, and key environment variables.

## Agent Coordination & Orchestration

**Agent Zero** – *Control-plane agent orchestrator.* Agent Zero coordinates the overall system as a supervisor and tool dispatcher[\[1\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L5-L8). It subscribes to the NATS event bus and orchestrates tasks across other agents (Archon, PMOVES.YT, media analyzers, etc.)[\[1\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L5-L8). **Path:** pmoves/services/agent-zero (with upstream code in submodule **PMOVES-Agent-Zero**). **Image:** Built from services/agent-zero/Dockerfile (published as agent-zero image via CI). **Ports:** Exposes **8080** (FastAPI supervisor API) and **8081** (Agent Zero’s UI, served from an embedded agent runtime)[\[2\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L6-L13)[\[3\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L602-L610). **Health:** GET /healthz on port 8080 returns the supervisor and embedded agent status[\[4\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L34-L42) (Agent Zero’s /healthz also includes NATS connectivity and, if running, the embedded runtime’s own health)[\[4\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L34-L42). The supervisor also provides endpoints like /config/environment (to inspect resolved config) and an /mcp/\* API for issuing *MCP commands* to the agent runtime[\[5\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L34-L43)[\[6\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L51-L59). **Env:** Key variables include PORT (default 8080\) for the API, NATS\_URL (NATS broker URI)[\[7\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L15-L19), and AGENT\_ZERO\_API\_BASE (address of the embedded agent runtime, default http://127.0.0.1:80)[\[8\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L16-L23). Agent Zero also takes integration endpoints: e.g. HIRAG\_URL (Hi-RAG v2 gateway URL), YT\_URL (PMOVES.YT service URL), RENDER\_WEBHOOK\_URL (ComfyUI webhook URL), OPEN\_NOTEBOOK\_API\_URL & token (Open Notebook API for knowledge base), etc.[\[9\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L23-L31). **Depends on:** nats (messaging). In docker-compose, it’s in the agents profile and ensures NATS is running before startup[\[10\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L578-L586)[\[11\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L597-L604).

**Archon** – *Supabase-driven agent service (vendor-provided).* Archon is an upstream integration (included via submodule **PMOVES-Archon**) that connects to the Supabase data stack to provide additional agent capabilities and UI. It manages agent “forms” and uses Supabase (Postgres \+ REST API) as a backing store for prompts and state[\[12\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/archon/README.md#L9-L17). **Path:** pmoves/services/archon (wrapper Dockerfile) and pmoves/integrations/archon (submodule with Archon source). The Docker build either clones the external repo or uses the submodule code[\[13\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/services/archon/Dockerfile#L81-L90). **Image:** built as archon (with optional UI image). **Ports:** Exposes **8091** for Archon’s API server and **8051/8052** for internal microservices (MCP and agent comm), plus an **Archon UI** served separately on port **3737**. In development, the Archon UI is built from the archon-ui-main submodule and served via an archon-ui container on port 3737[\[14\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.archon-ui.submodule.yml#L2-L10)[\[15\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.archon-ui.submodule.yml#L12-L16). The main Archon API listens on 8091 (mapped from container 8091\)[\[16\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L640-L649). **Health:** Archon provides /healthz (Docker healthcheck targets this) which checks both the service and its Supabase connectivity[\[17\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L20-L28)[\[18\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L648-L656). **Env:** Archon expects Supabase connection info: SUPABASE\_URL and SUPABASE\_SERVICE\_KEY (service role key) are mapped from the PMOVES environment (SUPA\_REST\_URL etc.)[\[12\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/archon/README.md#L9-L17). Variables like ARCHON\_SUPABASE\_BASE\_URL (base URL of Supabase REST, e.g. http://host.docker.internal:65421 in local dev) and SUPA\_REST\_URL are used to configure this[\[19\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/archon/README.md#L16-L24). Archon also uses MCP\_SERVICE\_URL, MCP\_CLIENT\_ID/SECRET for connecting to the Agent Zero MCP interface[\[20\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/services/archon/Dockerfile#L56-L64)[\[21\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/services/archon/Dockerfile#L70-L78). **Depends on:** Supabase (PostgREST) – Archon will wait for the Supabase REST endpoint (it injects SUPA\_REST\_URL to ensure it uses the correct host)[\[17\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L20-L28). It also connects to nats for event messaging[\[22\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L631-L639). Archon’s UI communicates with the Archon API (configured via env VITE\_API\_URL).

**Mesh Agent** – *Distributed node announcer.* The “mesh-agent” service runs without external API, periodically announcing the host’s presence/capabilities on NATS for multi-host orchestration. **Path:** pmoves/services/mesh-agent. **Image:** built from local Dockerfile. **Ports:** none (no HTTP interface). **Env:** NATS\_URL for broker connection and HIRAG\_URL (defaults to the Hi-RAG v2 GPU gateway)[\[23\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L660-L668). ANNOUNCE\_SEC (default 15s) controls how often it broadcasts a heartbeat on the event bus[\[23\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L660-L668). **Depends on:** NATS. (This helps coordinate multi-GPU or multi-node setups by registering nodes on the bus.)

**Channel Monitor** – *External content watcher.* The channel-monitor service tracks external content feeds (e.g. YouTube channels) and triggers ingestion jobs. **Path:** pmoves/services/channel-monitor. **Image:** built from local Dockerfile. **Ports:** **8097** (HTTP API for status/health). **Env:** CHANNEL\_MONITOR\_CONFIG\_PATH (JSON file listing channels to monitor), and CHANNEL\_MONITOR\_QUEUE\_URL (URL of PMOVES.YT’s ingest endpoint)[\[24\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L390-L398). For example, by default it posts to http://pmoves-yt:8077/yt/ingest to enqueue new videos[\[24\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L390-L398). It also uses a database (Supabase/Postgres) for state – CHANNEL\_MONITOR\_DATABASE\_URL is supplied via env file – and a namespace for tagging (CHANNEL\_MONITOR\_NAMESPACE)[\[24\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L390-L398). **Depends on:** postgres (for storing seen video IDs) and pmoves-yt (to ingest found content)[\[25\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L399-L406). Channel Monitor runs in the “yt” compose profile.

## Retrieval & Knowledge Services

**Hi-RAG Gateway (v1)** – *Original “Hybrid RAG” HTTP gateway.* This is the first-generation Retrieval-Augmented Generation service that combines vector search and a knowledge graph. It provides a unified **HTTP API on port 8086** (mapped to host **8089** for CPU mode) for querying knowledge with LLM assistance[\[26\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L8-L16). **Path:** pmoves/services/hi-rag-gateway (with an optional GPU variant). **Image:** built from local Dockerfile (hi-rag-gateway/Dockerfile). **Ports:** CPU container listens on 8086 and is exposed as host **8089**; the GPU variant (with CUDA and rerank enabled) also listens on 8086 but is exposed as **8090**[\[27\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L9-L17) (in compose, profile gpu+legacy)[\[26\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L8-L16). **Health/API:** It serves endpoints like POST /hirag/query for queries[\[28\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L28-L36). (A health check endpoint is not explicitly documented for v1, but logs indicate it now supports /healthz as well.) **Env:** Key environment vars include QDRANT\_URL and QDRANT\_COLLECTION (for vector DB), NEO4J\_URL/NEO4J\_USER/NEO4J\_PASSWORD (for the Neo4j graph)[\[29\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L13-L17), and toggles for optional features: USE\_MEILI with MEILI\_URL/MEILI\_API\_KEY to enable Meilisearch full-text augmentation[\[30\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L16-L19), and RERANK\_ENABLE plus model settings to enable cross-encoder re-ranking of results[\[31\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L18-L21). By default the CPU service has rerank off, GPU has it on (using a larger model)[\[32\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L20-L25). **Depends on:** qdrant (vector store) and neo4j – these are required data stores[\[33\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L11-L19). (Both are included in the docker-compose stack[\[33\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L11-L19).) The gateway uses those to fetch relevant context (vector similarity \+ graph traversal) which it then ranks/returns. *Note:* v1 is considered “legacy” but retained for baseline comparisons[\[34\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L38-L45).

**Hi-RAG Gateway v2** – *Next-gen RAG service with reranking and multimodal support.* This is the improved hybrid RAG gateway (experimental) which includes advanced features like end-to-end retrieval with cross-encoder reranking, optional multimodal embedding support, and integration with the “Geometry/CHIT” bus for structured outputs[\[34\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L38-L45). **Path:** pmoves/services/hi-rag-gateway-v2. **Image:** built from local Dockerfile(s) (hi-rag-gateway-v2/Dockerfile for CPU, plus a Dockerfile.gpu for a GPU-enabled build). **Ports:** Container listens on 8086; by default host maps **8086**→8086 for the CPU v2 service, and **8087**→8086 for the GPU v2 service (profile gpu)[\[35\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L563-L571)[\[36\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L564-L572). **API:** Similar query endpoints (e.g. /hirag/query) and a richer JSON output. It also exposes a /healthz endpoint and can emit telemetry events on the CHIT (geometry) bus. **Env:** Many tunables – it uses the same QDRANT\_URL/COLLECTION and MEILI\_URL as v1[\[37\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L416-L424)[\[38\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L426-L434). Rerank is **enabled by default** with model configurable (RERANK\_MODEL defaulting to BAAI/bge-reranker-base on CPU, or a larger Qwen model on GPU)[\[39\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L420-L428)[\[40\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L524-L532). It also accepts ALPHA (blend weight between vector and graph scores), GRAPH\_BOOST, and cache controls for entity linking[\[39\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L420-L428)[\[41\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L434-L442). V2 integrates with additional ML backends: if USE\_OLLAMA\_EMBED=true it will query a local Ollama LLM server for embeddings[\[38\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L426-L434), and it can optionally call a **TensorZero** embedding service (TENSORZERO\_BASE\_URL & API key) for advanced embedding or rerank tasks[\[42\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L430-L438)[\[43\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L552-L560). It also uses Supabase for real-time event broadcasting: SUPA\_REST\_URL and SUPABASE\_REALTIME\_URL are injected so that the gateway can publish to channels or use the Supabase listener[\[44\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L442-L450). **Depends on:** qdrant, neo4j, and (if enabled) meilisearch. It is tagged under compose profile “gateway” (in addition to “workers”) and can run concurrently with or instead of v1. *Note:* Hi-RAG v2 is the **preferred** retrieval path for PMOVES, intended to eventually replace v1 for most uses[\[34\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L38-L45).

**DeepResearch** – *LLM-based research planner integration.* DeepResearch is a worker service that connects PMOVES to Alibaba’s **Tongyi DeepResearch** model for automated research/summarization tasks[\[45\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L3-L7). It acts as a bridge between the event bus and the DeepResearch LLM: it listens on NATS for research.deepresearch.request.v1 events and produces ...result.v1 responses with summaries and references[\[45\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L3-L7)[\[46\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L49-L57). **Path:** pmoves/services/deepresearch (with upstream code brought in at build time – it clones Alibaba-NLP/DeepResearch during the image build to support local mode)[\[47\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L19-L27). **Image:** built from services/deepresearch/Dockerfile (published as deepresearch image if configured). **Ports:** No external API; however, it runs an internal health server. By default it opens **8098** for health monitoring (as set by DEEPRESEARCH\_HEALTH\_PORT)[\[48\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L689-L697)[\[49\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L691-L695). **Health:** The worker likely responds on http://localhost:8098/healthz (used for internal checks; not typically exposed beyond compose network). **Env:** DEEPRESEARCH\_MODE selects the execution mode: "openrouter" (default) uses the OpenRouter API to call an online Tongyi DeepResearch model, whereas "local" mode will use a self-hosted DeepResearch service if available[\[50\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L29-L37). Corresponding vars: OPENROUTER\_API\_KEY (needed for OpenRouter mode)[\[51\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L31-L38), DEEPRESEARCH\_OPENROUTER\_MODEL (model name, default "tongyi-deepresearch")[\[51\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L31-L38), and DEEPRESEARCH\_API\_BASE with DEEPRESEARCH\_PLANNING\_ENDPOINT (URL for a local planner’s API when in local mode)[\[52\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L31-L39). It also integrates with the Notebook service: if OPEN\_NOTEBOOK\_API\_URL and token are provided, DeepResearch will automatically publish its results into Open Notebook (creating a new “source” entry containing the summary and related data)[\[53\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L34-L42)[\[54\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L58-L66). It logs any Notebook publish failures in the result metadata for retry[\[54\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L58-L66). **Depends on:** nats (for receiving and emitting events). DeepResearch runs as part of the agents profile (alongside Agent Zero and Archon) and ensures NATS is up before starting[\[23\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L660-L668)[\[55\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L693-L699).

**SupaSerch** – *“Multimodal Holographic Deep Research” service (search aggregator).* SupaSerch (PMOVES-Supaserch) is a long-running, agent-assisted search service that orchestrates complex search queries using multiple tools[\[56\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L3-L10). It leverages LLM agents and indices to handle user research queries continuously (intended to run on a VM with internet access)[\[57\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L5-L10). Internally, SupaSerch coordinates several components: it can dispatch tasks to the DeepResearch worker (for large-language-model planning), call Archon/Agent Zero MCP tools (for code execution, web crawling, etc.), use the **CHIT Geometry Bus** for structured output representations, and query **Supabase/Qdrant/Meili** knowledge bases for information retrieval[\[57\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L5-L10). Results are then compiled and made available to users or other agents. **Path:** pmoves/services/supaserch. **Image:** named pmoves-supaserch[\[58\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L1-L9). CI publishes it (multi-arch) to GHCR and Docker Hub (a GitHub workflow builds pmoves-supaserch:pmoves-latest nightly)[\[59\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L46-L48). **Ports:** Default **8099** for HTTP. SupaSerch provides an HTTP interface (e.g. health and metrics) on this port[\[60\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L12-L20). **Health:** GET /healthz on 8099 returns {"status":"ok"} plus info on NATS connectivity and any configured fallback search endpoint[\[61\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L12-L19). It also exposes Prometheus metrics at /metrics on the same port[\[61\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L12-L19). **Env:** SUPASERCH\_PORT (container port, default 8099\)[\[62\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L16-L19) and SUPASERCH\_HOST\_PORT (to map it externally) configure the port. NATS\_URL is required for receiving search tasks and publishing results[\[63\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L18-L21). SupaSerch can use an optional HTTP fallback search API – set by SUPASERCH\_HTTP\_FALLBACK\_URL (with placeholders for {query}) – in case its internal pipeline fails; SUPASERCH\_FALLBACK\_TIMEOUT controls that call[\[64\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L18-L22). It also needs the knowledge store endpoints: SUPA\_REST\_URL for Supabase (to fetch index data) and HIRAG\_URL for the geometry/RAG gateway[\[65\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L20-L23). **Depends on:** nats (for its internal request/result envelope mechanism on subjects supaserch.request.v1 and .result.v1[\[66\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L50-L55)). In practice, Agent Zero or other clients publish queries to SupaSerch via NATS, and SupaSerch replies on the bus, using the various tools mentioned to compile answers.

**Open Notebook** – *Knowledge base / note-taking integration.* Open Notebook is an external knowledge repository (backed by SurrealDB, referred to in PMOVES docs) that PMOVES integrates with for persistent storage and search of notes, sources, and AI-generated content. It is included as a submodule **PMOVES-Open-Notebook**. In the PMOVES stack, Open Notebook provides an **API** (with its own auth token) that services like DeepResearch and notebook-sync use to publish and retrieve content. **Path:** External submodule (the core may run SurrealDB and a Node/Go backend, not detailed in this repo). **Image:** published as pmoves-open-notebook (assumed; built from submodule). **Ports:** Typically runs on its own port (not specified here, but likely default Surreal or API port). **Env:** PMOVES services expect OPEN\_NOTEBOOK\_API\_URL and OPEN\_NOTEBOOK\_API\_TOKEN to be set for connectivity[\[67\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L687-L694). For example, DeepResearch and notebook-sync will use these to POST new notes or query existing ones. **Depends on:** If deployed, it requires its backing DB (SurrealDB) – likely handled within the submodule. In PMOVES it’s treated as an external integration that the agents can use for knowledge persistence.

**Extract Worker** – *Text embedding & indexing service.* The extract-worker service ingests raw text (from documents or transcripts) and produces vector embeddings and search index entries. It also requires an embedding model name (SENTENCE\_MODEL, default all-MiniLM-L6-v2 for sentence embeddings)[\[78\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L198-L201). For storing results in Supabase, it uses SUPA\_REST\_URL (the PostgREST endpoint) – in local dev this is pointed to the Supabase CLI REST port on host.docker.internal[\[79\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L202-L205). **Depends on:** qdrant, meilisearch. Other services (PDF ingest, Notebook sync) call the extract-worker’s API to vectorize content.

**Jellyfin-AI Media Stack** – *Local media server integration.* **Jellyfin** is an open-source media server; in PMOVES it’s used to manage a media library that AI services can analyze. **Render Webhook** – *ComfyUI render callback handler.* This service integrates with **ComfyUI** (a Node-based Stable Diffusion workflow engine) by providing a secure webhook endpoint to receive rendered images or notifications. For instance, when an image is generated in ComfyUI (triggered by an agent via MCP), Comfy can send an HTTP POST to this service which will then handle the output (store it, mark a task complete, etc.). **Path:** pmoves/services/render-webhook. **Image:** built locally. **Ports:** **8085** for its HTTP interface[\[123\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L184-L188). **Env:** It needs to write data to Supabase (so that the system knows a render result is ready). Thus SUPA\_REST\_URL is provided (defaults to the Supabase PostgREST)[\[124\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L180-L188). It also has a DEFAULT\_NAMESPACE (e.g. “pmoves”) to categorize indexing of render outputs[\[124\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L180-L188). RENDER\_WEBHOOK\_SHARED\_SECRET is a secret token that ComfyUI must include – the service will verify this to ensure the request is authorized[\[124\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L180-L188). RENDER\_AUTO\_APPROVE (boolean) may indicate if the webhook should auto-acknowledge incoming renders as completed without manual review[\[125\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L182-L186). **Depends on:** Supabase (for writing records) and obviously an external ComfyUI workflow sending the webhooks. In practice, an agent (via Agent Zero) triggers ComfyUI to start a render with this service’s URL as the callback, and this service updates the PMOVES data (like uploading the image to MinIO and logging an event).

**Publisher-Discord** – *Discord notification bot.* This service listens on NATS for certain events (like new content ingested or summaries ready) and publishes them to a Discord channel via a webhook. **Path:** pmoves/services/publisher-discord. **Image:** built locally. **Ports:** **8094** on host mapped to container 8092 (the service itself doesn’t need to expose HTTP; the port mapping is likely just for an internal healthcheck or metrics)[\[126\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L748-L755). **Env:** DISCORD\_WEBHOOK\_URL – the Discord webhook endpoint where messages will be sent[\[127\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L748-L752). NATS\_URL for subscribing to events[\[127\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L748-L752). DISCORD\_SUBJECTS – a comma-separated list of NATS subjects to subscribe to[\[128\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L750-L752). It ensures NATS is running (depends\_on in compose)[\[129\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L751-L758). This service allows the PMOVES system to send notifications to a user’s Discord (for example, posting a summary of a video just processed, with a link to Jellyfin for playback[\[130\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/jellyfin-ai/README.md#L34-L42)).

## Core Data & Infrastructure Components

*(These are standard services used by the PMOVES platform; they are pulled as base images rather than built in the repo, but defined in the compose file and vital for full deployment.)*

**Supabase (Postgres \+ PostgREST)** – *Primary database and REST API.* PMOVES uses a Supabase stack for its central data store. In local deployments this consists of a Postgres database with the **pgvector** extension and a PostgREST service in front of it. **Path/Images:** Uses the image ankane/pgvector for Postgres (which comes with vector search capabilities) and postgrest/postgrest:latest for the REST API[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Ports:** PostgREST is exposed at **3010** (mapping container 3000\) for internal services[\[136\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L30-L35). The Postgres DB is on the default **5432** (mapped same)[\[137\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L2-L9). **Env:** The database is initialized with schemas and roles via scripts in supabase/initdb/\*.sql (which include PMOVES core schema, Archon prompt tables, etc.). PostgREST is configured with PGRST\_DB\_URI (points to the Postgres), PGRST\_DB\_SCHEMA (e.g. public,pmoves\_core), PGRST\_DB\_ANON\_ROLE=anon (for public selects), and JWT settings for Supabase auth tokens[\[138\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L26-L34). **Data:** The Supabase DB holds most persistent state: e.g. tables for content (videos, transcripts, etc.), user data for wger/firefly if shared, and Archon uses it to store prompt templates and agent forms. **Depends on:** – (PostgREST depends on Postgres). In dev, a Supabase CLI service might provide additional components (like gotrue, realtime websocket). In compose, a postgrest-cli service is defined for connecting to a dev Supabase CLI Postgres as well[\[139\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L50-L59). The PMOVES environment yields a unified REST endpoint (SUPA\_REST\_URL) which services use to talk to this data layer.

**Qdrant Vector DB** – *Semantic vector store.* Qdrant is used to store embedding vectors for documents, enabling semantic similarity search. **Image:** qdrant/qdrant:v1.10.0[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Path:** (No custom code, configured via env or default). **Ports:** **6333** for API (mapped to host 6333\)[\[140\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L66-L74). **Env:** Default (no auth by default). Collections are created for content chunks (e.g. pmoves\_chunks). Services like Hi-RAG and extract-worker are configured with QDRANT\_URL (pointing to http://qdrant:6333) and the collection name[\[78\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L198-L201). **Usage:** Hi-RAG queries Qdrant for top-N similar vectors for a query; extract-worker inserts new vectors on ingest.

**Neo4j Graph DB** – *Knowledge graph store.* Neo4j (v5.22) holds the concept/entity graph for Hi-RAG’s knowledge base[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Image:** neo4j:5.22 (official) with APOC if needed. **Ports:** **7474** (HTTP UI) and **7687** (Bolt protocol)[\[141\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L82-L90). **Env:** Auth is configured via NEO4J\_AUTH (e.g. neo4j/password from env)[\[142\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L84-L91). The compose mounts cypher scripts (in neo4j/cypher) to bootstrap the graph with indices and constraints. **Usage:** Hi-RAG uses Neo4j to store and retrieve entities and their relationships (allowing the system to do things like link videos to topics, etc.). Queries from Hi-RAG combine Qdrant results with Neo4j traversals (“graph booster”). The gateway’s env includes NEO4J\_URL, NEO4J\_USER, NEO4J\_PASSWORD for connecting to this DB[\[29\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L13-L17).

**Meilisearch** – *Full-text search engine.* Meilisearch (v1.8) is optionally used to store text for keyword search[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Image:** getmeili/meilisearch:v1.8[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Ports:** **7700** for HTTP API[\[143\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L70-L78). **Env:** MEILI\_MASTER\_KEY (set via env files) to secure the API[\[144\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L74-L81). **Usage:** If USE\_MEILI=true in Hi-RAG, the gateway will query Meili for keyword matches to complement vector results[\[145\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/hi-rag-gateway/README.md#L16-L20). The extract-worker also indexes text into Meili (it has MEILI\_URL and MEILI\_API\_KEY in env)[\[146\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L198-L205). Meili provides substring and typo-tolerant search which can boost recall for factual queries.

**MinIO** – *Object storage (S3-compatible).* MinIO stores large binary assets: downloaded videos, audio files, images, etc., so that other services can access them via presigned URLs. **Image:** minio/minio:latest. **Ports:** **9000** for S3 API, **9001** for the web console[\[147\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L100-L108). **Env:** MINIO\_ROOT\_USER/MINIO\_ROOT\_PASSWORD (set to default dev creds)[\[148\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L102-L106). In compose it’s run with the command server /data \--console-address ":9001" to enable the UI[\[147\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L100-L108). **Usage:** All ingest and analysis services use MinIO as the central file repository. For example, PMOVES.YT saves video files to MinIO (bucket “assets”), FFmpeg-Whisper reads the file from MinIO to transcribe it, and the transcript might be saved back (or sent to Supabase as text). Similarly, when an image is generated via ComfyUI, it can be uploaded to MinIO and then a presigned URL is generated for Discord or Archon to access it. **Depends on:** – (backed by a volume for persistence). The Presign service and others share the MINIO\_\* connection settings to interface with it.

**NATS** – *Message bus.* NATS (v2.10) is the core event broker for agent communication[\[131\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L12-L15). **Image:** nats:2.10-alpine. **Ports:** **4222** (NATS client port)[\[149\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L579-L587). **Env:** in compose, runs with \-js enabled to allow JetStream (persisted topics)[\[149\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L579-L587). No auth in dev (can be secured in prod if needed). **Usage:** Many services use NATS to decouple workflows. For instance, Agent Zero publishes commands and receives results via NATS subjects, DeepResearch listens for research requests on a subject and responds on another[\[45\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L3-L7)[\[46\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/deepresearch/README.md#L49-L57), Publisher-Discord listens for various ingest.\* subjects to push notifications, etc. Agent Zero’s MCP interface can use NATS (JetStream) for reliable command delivery (controlled by AGENTZERO\_JETSTREAM=true)[\[150\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/agent-zero/README.md#L18-L22). **Depends on:** – (just runs standalone). NATS is started in the agents profile before Agent Zero, Archon, etc.[\[10\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docker-compose.yml#L578-L586). It’s essential for the loosely coupled multi-agent design of PMOVES.

**Monitoring Dependencies:** (For completeness) **Prometheus** scrapes metrics (no incoming ports exposed publicly), **Grafana** on port 3000 (with a datasource to Prometheus and Loki), **Loki** on port 3100, **Promtail** runs on each node to ship logs to Loki, and **cAdvisor** on port 8080 to provide container stats to Prometheus. These ensure all custom services export /metrics and /healthz which are checked regularly[\[151\]](https://github.com/POWERFULMOVES/PMOVES.AI/blob/45caf6783bb14f1f4e357c5465d7bfeb76730fb1/pmoves/docs/services/supaserch/README.md#L26-L34)[\[134\]](https://github.com/POWERFULMOVES/PMOVES-BoTZ/blob/28f16227d711898ca7ea6543ab6119ae82a8350a/docs/PMOVES.AI-Edition-Hardened.md#L54-L58).
